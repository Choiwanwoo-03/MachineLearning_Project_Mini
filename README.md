# 🏢 아파트 실거래가 추정 모델 개발 프로젝트
**Apartment Real Transaction Price Estimation Model**

본 프로젝트는 머신러닝 알고리즘을 활용하여 아파트의 현재 실거래가를 정교하게 추정하고, 가격 결정에 영향을 미치는 주요 변수들을 분석하는 것을 목표로 합니다.

---

## 🎯 1. 프로젝트 목적 및 목표
* **실거래가 추정:** 공공데이터 및 입지 데이터를 결합하여 특정 시점의 아파트 매매가를 고도로 정밀하게 예측합니다.
* **데이터 최적화:** 결측치 처리, 이상치 제거 및 피처 엔지니어링을 통해 모델의 일반화 성능을 극대화합니다.
* **기술적 가이드 제공:** 데이터 수집부터 모델 평가까지 전 과정에 걸쳐 체계적인 머신러닝 파이프라인을 구축합니다.

## 📊 2. 데이터셋 구성
프로젝트의 데이터 아키텍처는 `데이터셋 종류.xlsx` 파일을 기준으로 설계되었습니다.
* **주요 데이터원:** 국토교통부 아파트매매 실거래 상세 자료 (공공데이터포털 API)
* **독립 변수 (Features):**
  - **단지 특성:** 전용면적, 층수, 건축년도, 총 세대 수 등
  - **입지 조건:** 지하철역/학교/대형마트와의 거리 (Kakao Map API 활용)
  - **거시 경제:** 기준 금리, 환율 변화 추이 등
* **종속 변수 (Target):** 아파트 실거래가 (매매가)

## 🛠 3. 기술 스택 (Tech Stack)
* **언어:** Python 3.10+
* **라이브러리:** 
  - 데이터 처리: Pandas, NumPy
  - 시각화: Matplotlib, Seaborn
  - 머신러닝: Scikit-learn, XGBoost, LightGBM
* **환경:** VS Code, Git/GitHub

## 📈 4. 모델 평가 지표
부동산 가격 예측의 특성을 고려하여 아래 지표를 통해 모델의 성능을 다각도로 검증합니다.
* **RMSE (Root Mean Square Error):** 오차의 크기를 직관적으로 파악
* **MAE (Mean Absolute Error):** 실제 가격과의 절대적 차이 측정
* **MAPE (Mean Absolute Percentage Error):** 가격 규모 대비 오차 비율 확인 (가장 권장되는 지표)

## 📁 5. 저장소 구조
- `data/`: 원본 및 전처리 데이터 (`raw`, `processed`)
- `notebooks/`: EDA(탐색적 데이터 분석) 및 실험용 Jupyter Notebook
- `src/`: 전처리, 학습, 추론을 위한 모듈형 파이썬 스크립트
- `models/`: 학습이 완료된 모델 저장 파일 (.pkl, .h5)

## 📁 6. 상세 폴더 구조 및 용도 (Directory Details)

조원들은 각 파일의 성격에 맞는 폴더에 결과물을 저장하여 협업 효율을 유지해야 합니다.

| 폴더명 | 용도 및 설명 | 주요 포함 파일 예시 |
| :--- | :--- | :--- |
| **`data/raw/`** | 수집된 원본 데이터를 보관합니다. (수정 금지) | `데이터셋 종류.xlsx`, 공공데이터 API 원본 |
| **`data/processed/`** | 전처리 및 피처 엔지니어링이 완료된 학습용 데이터입니다. |  |
| **`data/external/`** | 모델 성능 향상을 위한 외부 지표를 보관합니다. | 금리 데이터, 환율 변동 내역(JPY/KRW 등) |
| **`notebooks/`** | 실험적인 EDA(탐색적 데이터 분석) 및 시각화용 파일입니다. |  |
| **`src/`** | 재사용 가능한 모듈형 파이썬 스크립트를 관리합니다. |  |
| **`models/`** | 학습이 완료된 모델 파일 스냅샷을 저장합니다. |  |
| **`config/`** | 하이퍼파라미터 설정 및 API 인증 키 등을 관리합니다. |  |
| **`reports/`** | 최종 분석 결과 보고서 및 시각화 그래프를 보관합니다. |  |
